# 错误码规范

## 概述

采用 **64 位结构化错误码**设计，通过 **Crockford Base32 编码**呈现为易于人工识别的 **17 字符可读格式**（包含 4 个分隔符）。

**格式示例：** `B0-AAAC-AAB-AAB-C`

> [!NOTE]
> 64 位固定长度能够在一个空间内承载版本、类别、系统、模块、序列与校验码等多个维度的信息，兼顾了容量、可扩展性与跨系统的全局唯一性。

---

## 错误码结构

### 位段划分

```
┌──────────┬───────────┬────────────┬───────────┬───────────┬───────────┐
│ Version  │ Category  │  System    │  Module   │    Seq    │    CRC    │
│  4 bits  │  5 bits   │  20 bits   │  15 bits  │  15 bits  │   5 bits  │
├──────────┼───────────┼────────────┼───────────┼───────────┼───────────┤
│ Bit63-60 │ Bit59-55  │  Bit54-35  │  Bit34-20 │  Bit19-5  │  Bit4-0   │
└──────────┴───────────┴────────────┴───────────┴───────────┴───────────┘
```

### 字段说明

| 字段         | 位数    | 容量      | 说明                        |
| ------------ | ------- | --------- | --------------------------- |
| **Version**  | 4 bits  | 16        | 版本号，用于协议升级兼容    |
| **Category** | 5 bits  | 32        | 错误性质，快速区分错误类型  |
| **System**   | 20 bits | 约 104 万 | 系统 ID，标识错误来源系统   |
| **Module**   | 15 bits | 约 3.2 万 | 模块 ID，定位系统内部逻辑块 |
| **Seq**      | 15 bits | 约 3.2 万 | 序列号，具体错误点标识      |
| **CRC**      | 5 bits  | 32        | 校验位，防止手抄/传输错误   |

---

## 可读格式

错误码使用 **Crockford Base32 编码**，采用 **错位拼接** 设计，使用中线分隔。为了便于快速识别错误类型，Category 被前置至首位。

### 编码设计

- 原始位序：Version (4 bits) → Category (5 bits) → System (20 bits) → Module (15 bits) → Seq (15 bits) → CRC (5 bits)
- **编码后位序**：Category (5 bits) → Version (4 bits + 1 bit padding) → System (20 bits) → Module (15 bits) → Seq (15 bits) → CRC (5 bits)
- 通过重新排序，使得错误类别字符首先出现，便于快速视觉识别

### 格式示例

```
B0-AAAC-AAB-AAB-C
│  │    │   │   │
│  │    │   │   └─ CRC：校验位，防止错误码被破坏或篡改
│  │    │   └────── Seq：序列号，该模块内的具体错误点
│  │    └────────── Module：模块 ID，系统内的功能模块划分
│  └─────────────── System：系统 ID，错误来源的具体系统
└────────────────── Category + Version：错误类别 + 版本
                    (首字符 B=业务、S=系统、A=认证、V=校验等)
```

### 示例

| 错误码              | 首字符 | 含义                      |
| ------------------- | ------ | ------------------------- |
| `S0-AAAC-AAC-AAB-Q` | S      | 系统故障 (SYSTEM)         |
| `B0-AAAC-AAB-AAB-C` | B      | 业务异常 (BUSINESS)       |
| `V0-AAAC-AAD-AAB-Q` | V      | 参数校验错误 (VALIDATION) |
| `A0-AAAC-AAE-AAB-C` | A      | 认证错误 (AUTH)           |

## 错误性质分类 (Category)

| 首字符 | 值  | 名称          | 说明                                                            |
| ------ | --- | ------------- | --------------------------------------------------------------- |
| S      | 25  | `SYSTEM`      | 系统级故障（数据库断连、空指针、内存溢出），必报警              |
| B      | 11  | `BUSINESS`    | 业务逻辑异常（余额不足、订单状态不符、活动已过期），不报警      |
| V      | 27  | `VALIDATION`  | 用户行为错误（参数格式错、验证码填错、非法操作），前端/用户负责 |
| T      | 26  | `THIRD_PARTY` | 外部依赖异常（第三方支付超时、外部 API 故障），触发熔断         |
| A      | 10  | `AUTH`        | 认证与安全（Token 失效、签名校验失败、恶意请求），安全审计关注  |
| R      | 24  | `RATE_LIMIT`  | 流量限制（触发限流、QPS 超限）                                  |
| D      | 13  | `DEGRADE`     | 降级/熔断（功能已被手动或自动下线）                             |
| C      | 12  | `CONFLICT`    | 数据冲突（并发修改导致的锁冲突），建议调用方稍后重试            |
| F      | 15  | `FEATURE`     | 功能不可用（地区限制/App 版本过低/功能下线等）                  |

### 保留编号

**Module = `000`** 为保留编号，用于定义通用错误，业务系统不可使用。

| 错误码             | 名称                   | 说明                                   |
| ------------------ | ---------------------- | -------------------------------------- |
| `S0-***-000-500-*` | Internal Server Error  | 系统内部未知错误，代码崩溃的兜底错误码 |
| `A0-***-000-401-*` | Unauthorized           | 未登录或者 Token 失效                  |
| `A0-***-000-403-*` | Forbidden              | 已登陆但无权访问                       |
| `B0-***-000-404-*` | Not Founded            | 资源不存在                             |
| `U0-***-000-415-*` | Unsupported Media Type | 无法解析请求体或请求体格式错误         |
| `T0-***-000-504-*` | Gateway Timeout        | 调用第三方服务超时                     |
